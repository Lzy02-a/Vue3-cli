# AGENTS.md

> 本文档用于约束 AI Agent 在本项目中的工作方式。每次开始处理用户请求前，应先阅读并遵守本文档。

## 1. 项目概览

- 项目类型：Vue 3 + TypeScript + Vite
- 包管理器：按项目现有 lockfile 或用户指定使用
- 主要目录：
  - `src/api/`：接口请求封装
  - `src/pages/`：页面组件
  - `src/utils/`：通用工具、请求实例等
  - `src/router/`：路由配置
  - `src/stores/`：Pinia 状态管理

## 2. 每次任务开始前

1. 先阅读当前任务涉及的文件。
2. 如果任务涉及代码修改，先理解现有风格和依赖关系。
3. 不确定需求时先提问，不要自行假设关键业务逻辑。
4. 修改前避免覆盖用户未提交或未说明的改动。
5. 优先做最小必要修改，不引入无关重构。

## 3. 代码修改规范

- 使用 TypeScript 时保持类型明确，避免滥用 `any`。
- Vue 文件优先使用 `<script setup lang="ts">`。
- API 请求统一从 `src/api/` 发起，Axios 实例统一从 `src/utils/instance.ts` 引入。
- vue 文件样式优先使用 Sass 样式
- 保持导入路径风格一致，项目内路径优先使用 `@/` 别名。
- 不随意新增依赖；确需新增时说明原因。
- 不主动创建无关文档、示例文件或测试文件，除非用户要求。

## 4. UI/UX 规范

- 页面应保持简洁、现代、响应式。
- 表单需要清晰的标签、占位提示、错误提示和交互状态。
- 按钮、输入框等交互元素要有 hover / focus / disabled 状态。
- 避免过度动画，保证可读性和可访问性。

## 5. 验证要求

代码修改后应尽量进行验证：

1. 优先检查相关文件的类型或 lint 诊断。
2. 如果改动较大，运行项目已有脚本，例如：
   - `npm run build`
   - `npm run format:check`
3. 如果验证命令失败，应说明失败原因和是否与本次修改有关。

## 6. 回复要求

- 回复保持简洁，先说结果，再说关键变更。
- 如果修改了文件，说明修改了哪些文件和原因。
- 如果未能验证，明确说明阻塞原因。
- 不输出大段代码，除非用户明确要求。

## 7. 禁止事项

- 不执行破坏性 Git 操作，例如 hard reset、force push，除非用户明确要求。
- 不擅自提交 commit 或 push。
- 不删除用户文件，除非用户明确要求。
- 不把密钥、token、密码等敏感信息写入代码或文档。
